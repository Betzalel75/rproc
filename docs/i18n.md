# Adding a new language to rproc

This guide explains how to add a new translation to rproc. The i18n system is
designed to be modular — one file per language, zero runtime allocations, a
single `match` arm to register.

Estimated time: 20–30 minutes once you know the steps.

---

## Step-by-step

### 1. Create the translation file

Copy `src/i18n/en.rs` as a starting point:

```bash
cp src/i18n/en.rs src/i18n/xx.rs
```

Replace `xx` with your language's [ISO 639-1 code][iso639] (e.g. `es` for
Spanish, `de` for German, `ja` for Japanese).

### 2. Translate every field

Open `src/i18n/xx.rs`. It starts with:

```rust
use super::Messages;

pub const MSG: Messages = Messages {
    // -- Sidebar ------------------------------------------------------------
    tab_processes: "Processes",
    tab_performance: "Performance",
    // … 150+ more fields …
};
```

Go through **every** field and replace the English string with your
translation. See the [field reference](#field-reference) below for context
about each string.

**Important rules:**

- Keep `{n}`, `{total}`, `{name}`, `{i}`, `{vendor}` and `{}` placeholders
  **exactly as they are** — they are replaced at runtime by the code.
- Use `\u{a0}` for non-breaking spaces (French thin space before colons, etc.).
- Use `\u{2026}` for the ellipsis character `…`.
- Use `\u{2014}` for the em-dash `—`.
- Use `\u{2197}` for the north-east arrow `↗`.
- Field names are lowercase English identifiers. Don't translate them.
- The file must compile — every field in the `Messages` struct must be present.

### 3. Register the language

Open `src/i18n/mod.rs`. You need to make four small changes:

**a) Declare the module** (near the top, next to `mod en; mod fr;`):

```rust
mod en;
mod fr;
mod xx;   // ← add this
```

**b) Add a variant to the `Lang` enum** (increment the discriminant):

```rust
pub enum Lang {
    En = 0,
    Fr = 1,
    Xx = 2,   // ← add this (use the actual language name)
}
```

**c) Add `as_str()` and `from_str()` cases**:

```rust
impl Lang {
    pub fn as_str(self) -> &'static str {
        match self {
            Lang::En => "en",
            Lang::Fr => "fr",
            Lang::Xx => "xx",   // ← ISO 639-1 code
        }
    }

    pub fn from_str(s: &str) -> Option<Self> {
        match s.trim().to_lowercase().as_str() {
            "en" | "english" => Some(Lang::En),
            "fr" | "french" | "français" | "francais" => Some(Lang::Fr),
            "xx" | "yourlang" => Some(Lang::Xx),   // ← add aliases
            _ => None,
        }
    }
}
```

**d) Add the message mapping**:

```rust
fn messages(lang: Lang) -> &'static Messages {
    match lang {
        Lang::En => &en::MSG,
        Lang::Fr => &fr::MSG,
        Lang::Xx => &xx::MSG,   // ← add this
    }
}
```

Also update the `current()` function's fallback arm if your language isn't 1:

```rust
pub fn current() -> Lang {
    match LANG.load(Ordering::Relaxed) {
        0 => Lang::En,
        1 => Lang::Fr,
        _ => Lang::Xx,   // ← if you want your language as default for unknown values
    }
}
```

### 4. Add the selector in Settings

Open `src/ui/settings.rs`. In the language selector card, add your language to
the array:

```rust
ui.horizontal(|ui| {
    for (choice, label) in [
        (Lang::En, "English"),
        (Lang::Fr, "Français"),
        (Lang::Xx, "Your Language"),   // ← add this
    ] {
        let selected = current == choice;
        // …
    }
});
```

### 5. Test

```bash
cargo build
cargo test
```

All 76+ tests must pass. If the build fails, check that every field in
`Messages` is present in your `xx.rs` file — Rust won't let you miss one.

Launch the app and switch to your language in **Settings → Language**.
